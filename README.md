# 持株会シミュレーター（株式会社良品計画 / 証券コード 7453）

社内向けの持株会シミュレーションツールです。基本は単体の `index.html` だけで動作します
（インストール・会員登録は不要）。スマホ・PCの両方に対応しています。

株価・配当の初期値は、**APIキー不要**の仕組みで取得した市場データ（前営業日終値ベース）を
自動で読み込みます（後述）。

## 使い方

- `index.html` をブラウザで開く、または公開URL（下記）にアクセスするだけ。
- 入力を変えると即時に再計算されます。
- 「初期値にリセット」で標準条件（市場データがあればその値）に戻せます。

## 公開URL（スマホ可・ログイン不要）

- かんたん表示（設定不要）:
  `https://htmlpreview.github.io/?https://raw.githubusercontent.com/Noguchi-tech/esop-simulator/main/index.html`
- 正式URL（GitHub Pagesを有効化後）:
  `https://noguchi-tech.github.io/esop-simulator/`
  （Settings → Pages → Branch: `main` / フォルダ `/(root)` で有効化）

> 公開手順の詳細は `docs/スマホ用URL公開手順.md` を参照してください。

## できること

- 毎月の拠出額、拠出年数・運用年数（初期値10年）の設定
- ボーナス拠出 あり／なし（ありの場合、6月・12月に毎月拠出額の2倍を自動で追加拠出）
- 年平均成長率（3% / 5% / 7% / カスタム）
- **配当の再投資を反映（任意・初期はON）**。1株あたり年間配当は市場データを初期値にしつつ手入力で変更可。
  配当は**野村口座のみ**に再投資され、退職金ポイント口座には拠出されません。
  **再投資は拠出期間中のみ**で、退職後は行いません。
- 退職金ポイント口座の**信託簿価を任意入力**（現在値 1,121円、将来を見据えた値も入力可）
- 信託簿価の更新方式（固定／一定周期で時価更新。更新周期の初期値は**5年ごと**）
- 退職後は退職金ポイントを時価評価し「特定口座移管分」として表示
- 結果サマリー、年次推移表（左から 年数／合計評価額／想定株価…）、グラフ
- 保有状況・配当・退職金ポイントの各設定は折りたたみ（必要な人だけ開く／初期は閉じた状態）

## 市場データの自動反映（APIキーは公開リポジトリに置きません）

「Web上の株価・配当をできるだけ反映したい。でも公開リポジトリにAPIキーは置きたくない」を両立する構成です。

- GitHub Actions（`.github/workflows/update-quote.yml`）が定期的に
  Yahoo Finance の**公開エンドポイント（キー不要）**から 7453 の株価・配当を取得し、
  `data/quote.json` を更新・コミットします。
- ページ側（`index.html`）は `data/quote.json` を読み込み、株価・配当の**初期値**として表示します
  （手入力で上書き可。リセットで市場データに戻る）。
- **APIキーは不要**です。仮に将来キー付きの有料データに切り替える場合も、キーはコードではなく
  GitHub の **Secrets** に保存し、ワークフロー内の環境変数から読むだけにします（リポジトリには出ません）。
- 取得に失敗しても、`index.html` 内のフォールバック値で問題なく動作します。

> 「リアルタイム」の厳密な約定価格ではなく、**前営業日終値ベースの近似**です（静的サイトの制約）。
> 仕組みの詳細は `docs/データ自動更新の仕組み.md` を参照してください。

## 計算の前提（すべて月次計算）

- 月次成長率 ＝ (1+年率)^(1/12) − 1
- 野村口座：毎月の拠出額 ÷ その月の株価で株数を積み増し、評価額＝株数 × 株価
- 配当（任意）：配当利回り＝初期1株配当 ÷ 初期株価とみなし、毎月 株数 × 株価 × 利回り/12 を
  **野村口座の株として再投資**（買い増し）。**再投資は拠出期間中のみ**で退職後は行いません。
  退職金ポイント口座には配当を付与しません。
- 退職金ポイント：付与ポイント ＝ 拠出額 × 50%（固定）÷ 信託簿価（既定 1,121円・任意入力可）
- 退職（拠出年数の終了）以降は新規拠出なし。退職金ポイントは退職時の時価で評価され、
  以後は特定口座移管分として株価成長に連動
- 「現在の野村口座」は**保有株数（株・小数2桁まで）**で入力。評価益の目安では、既存株は現在株価で
  取得したものとみなして基準額に算入します。

## 同梱ファイル

- `index.html` … ツール本体
- `口頭説明用スクリプト.md` … 対面説明用のカンペ（説明者向け）
- `data/quote.json` … 市場データ（自動更新される初期値）
- `scripts/fetch-quote.mjs` … 株価・配当の取得スクリプト（キー不要）
- `.github/workflows/update-quote.yml` … 市場データを定期更新するGitHub Actions
- `docs/スマホ用URL公開手順.md` … URL公開の手順
- `docs/データ自動更新の仕組み.md` … 市場データ自動取得の仕組みと運用

## ご注意

税金・手数料・制度変更・株式分割は考慮しない概算です（配当は任意で再投資の概算として反映できます）。
株価・配当の初期値は市場データの自動取得値ですが、リアルタイムの正確性・約定価格を保証するものでは
ありません。実際の受取額や課税関係は、会社制度・証券会社・税務上の扱いにより異なります。
投資判断は本人の責任で行ってください。
