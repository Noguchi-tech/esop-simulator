# 市場データ自動更新の仕組み（APIキーを公開リポジトリに置かない）

## 解決したい課題

- Web上の株価・配当を、できるだけ最新の値でツールに反映したい。
- でも、**公開リポジトリにAPIキー（シークレット）を置きたくない**。

## 結論：置かなくてできます

このリポジトリは「ビルド時（CI）に取得 → 静的JSONとして配信」という構成にしています。
ブラウザから直接 外部APIを叩くのではなく、**GitHub Actions が取得して `data/quote.json` を更新**し、
ページはその JSON を読むだけにしています。

```
[GitHub Actions（定期実行）]
        │  Yahoo Finance 公開エンドポイント（キー不要）
        ▼
   scripts/fetch-quote.mjs  ── 株価・直近1年の配当合計を取得
        │
        ▼
   data/quote.json を更新・コミット
        │
        ▼
[index.html] data/quote.json を fetch して初期値に反映（手入力で上書き可）
```

### なぜブラウザから直接 外部APIを叩かないのか

- 多くの株価APIは**CORS**を許可しておらず、ブラウザからの直接取得がブロックされます。
- キー付きAPIをブラウザで使うと、**キーが利用者に丸見え**になります（公開リポジトリ＝公開ページ）。
- 一方 `raw.githubusercontent.com` と GitHub Pages は CORS 許可があり、`data/quote.json` を
  安全に読めます。だから「CI で取得 → JSON 配信」が最もシンプルで安全です。

## 使用データソース

- 既定では **Yahoo Finance の chart エンドポイント（キー不要）** を使用：
  `https://query1.finance.yahoo.com/v8/finance/chart/7453.T?range=2y&interval=1d&events=div`
  - 現在値：`chart.result[0].meta.regularMarketPrice`
  - 配当：`chart.result[0].events.dividends` の直近365日分を合計＝年間配当（1株あたり）
- 取得失敗時は **既存の `data/quote.json` を維持**（CIは赤くしない）。

## 運用

- ワークフロー：`.github/workflows/update-quote.yml`
  - `schedule`：毎営業日 07:00 UTC（＝16:00 JST、東証クローズ後）
  - `workflow_dispatch`：Actions タブから手動実行も可能
- **重要**：`schedule`（定期実行）は **デフォルトブランチ（main）にワークフローがある場合のみ**動きます。
  feature ブランチで作業中は自動実行されません。`main` にマージすると自動更新が始まります。
  それまでは手動実行（Run workflow）か、`data/quote.json` のシード値が使われます。

## もし将来「キーが必要な有料データ」に切り替えるなら

公開リポジトリにキーを置かずに切り替えられます。

1. GitHub の **Settings → Secrets and variables → Actions** にキーを登録（例：`QUOTE_API_KEY`）。
2. ワークフローでそのシークレットを環境変数として渡す：

   ```yaml
   - name: Fetch quote & dividend
     run: node scripts/fetch-quote.mjs
     env:
       QUOTE_API_KEY: ${{ secrets.QUOTE_API_KEY }}
   ```

3. スクリプト側は `process.env.QUOTE_API_KEY` から読むだけにする。
   → キーはコードにも公開ページにも出ません。

## 注意点

- これは「**前営業日終値ベースの近似**」であり、ザラ場のリアルタイム約定価格ではありません
  （静的サイトの制約）。
- 配当は税引き前の概算です。株式分割・記念配当などで一時的に値がぶれることがあります。
